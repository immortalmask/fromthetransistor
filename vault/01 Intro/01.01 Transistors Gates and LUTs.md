# Transistors Gates and LUTs

## Why this block exists

The course promises a path “from the transistor,” but fabricating a transistor would produce no reusable component for the later browser. This block earns the abstraction jump: transistor switches justify logic levels; gates define Boolean functions; feedback and clocks justify storage; FPGA LUTs and registers provide the programmable fabric used by RTL.

## Lecture

A MOS transistor is useful here as a controlled switch, not as a device-physics project. Networks of switches restore and combine logic levels into gates. Gates implement Boolean functions. A multiplexer chooses one input by select bits; recursively, muxes can implement arbitrary lookup tables.

A two-input LUT stores four result bits. The course indexes them as:

```text
index = a * 2 + b
output = (mask >> index) & 1
```

Under this convention, mask `8` (`1000₂`) is AND, `6` (`0110₂`) is XOR, and `14` (`1110₂`) is OR. The convention matters: reversing input or bit order programs a different physical truth table even when the hex mask looks plausible.

A LUT is combinational. A counter requires stored state. At cycle zero the reset/current value is observed; each rising edge commits the incremented value; a `BITS`-wide state wraps modulo `2^BITS`. This small model is the bridge from ordinary C state-transition code to later synchronous RTL.

Real transistors have analog thresholds, delay, power, metastability, and manufacturing variation. The digital model deliberately hides those facts. Do not infer electrical safety, timing closure, or voltage compatibility from a logic simulation.

## Reverse-engineering lens

Given only four observed input/output rows, reconstruct the LUT mask under the known indexing rule. If the rule is unknown, test competing input/bit orders. This is a miniature version of inferring an undocumented decode table from controlled probes: coverage of the complete input space turns a guess into a strong model.
