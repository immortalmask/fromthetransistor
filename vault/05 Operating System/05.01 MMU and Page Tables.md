# 05.01 MMU and Page Tables

## Why this block exists

Processes need the illusion of private, contiguous memory even when physical RAM is shared and fragmented. The MMU makes that illusion enforceable by translating addresses and rejecting accesses that violate permissions.

## Lecture

### Mental model

Split a virtual address into virtual page number and page offset. A page-table walk uses one or more VPN fields to locate entries containing validity, physical page number, and access permissions. The offset is never translated. A TLB caches completed translations together with permission and address-space identity; it accelerates the same decision and must not change its meaning.

Define translation as a pure reference function first: input address, access kind, privilege, address-space ID, and page-table memory produce either physical address or a precise fault. Then add a finite TLB and compare every result against the reference.

### Invariants

- TLB hit and page-table walk return the same address or fault for the same state.
- User access never reaches supervisor-only or unmapped pages.
- Physical frame bases and page-table pointers satisfy page alignment and RAM bounds.
- Permission checks apply to the requested operation, including execute separately from read.
- Page-table changes become visible only after the required invalidation rule.

### Safety and limits

Run the core as a software model over an allocated byte array. Reject malformed entries and out-of-RAM table addresses before dereference. The initial model omits speculative execution, cache coherence, copy-on-write, huge pages, and side-channel defenses; none of those omissions permits bypassing the documented permissions.

## Workbook

For a supplied two-level table and six virtual accesses, predict every index, entry address, hit/miss, physical address, referenced/dirty update, and fault. Include a TLB of two entries with a stated replacement rule. Then change one PTE without invalidating and predict the stale result before testing it.

Draw the bit partition for two page sizes and explain which bits move between VPN and offset. Calculate internal fragmentation for the same mappings.

## Problem set

1. Translate four virtual addresses by hand through a two-level table, including an invalid first-level entry and a write to read-only memory.
2. Show how omitting an address-space identifier can expose one process's cached translation to another after a context switch.
3. Design precise fault records for not-present, malformed table, privilege, write, execute, and misaligned root failures.
4. Compare flush-all, per-address invalidation, and ASID-tagged TLBs for correctness and cost.
5. Give a checked arithmetic sequence for calculating a PTE address without overflow or out-of-RAM access.
6. Explain why setting a writable PTE and later clearing it requires invalidation even if physical memory did not move.

## Reverse-engineering lens
