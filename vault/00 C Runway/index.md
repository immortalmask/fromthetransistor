# 00 C Runway

The runway is part of the machine, not remedial decoration. Later layers deliberately remove host safety nets, so byte widths, pointer bounds, ownership, ABI behavior, and debugging discipline must already be usable.

## Modules

1. [[00.01 Course Orientation]] — scope, evidence, commands, and recovery.
2. [[00.02 Shell Build and Debug Loop]] — repeatable compilation, tests, sanitizers, and debugger use.
3. [[00.03 C Values and Control Flow]] — types, expressions, functions, and checked input.
4. [[00.04 Bytes Bits and Representation]] — widths, masks, shifts, signedness, and endianness.
5. [[00.05 Pointers Arrays and Strings]] — object bounds and length-aware traversal.
6. [[00.06 Structs Memory and Ownership]] — layout, resources, lifetime contracts, and cleanup.
7. [[00.07 Functions ABI and Object Files]] — calls, stack frames, symbols, relocations, and executables.
8. [[00.08 Digital Logic and State Machines]] — truth tables, registers, clocks, and cycle reasoning.

## Section artifact

A small evidence bundle containing a warning-clean C utility, decoded binary data, an annotated call/object trace, and a state-machine trace. Pass `gate-00` before depending on the runway under processor or kernel pressure.
