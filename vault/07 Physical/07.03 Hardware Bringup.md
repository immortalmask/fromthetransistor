# 07.03 Hardware Bringup

## Why this block exists

Bringup converts an unverified assembly into evidence one dependency at a time. A disciplined sequence prevents a late UART symptom from being blamed on software when power, clock, reset, configuration, or memory was never proven.

## Lecture

### Mental model

Use a ladder whose next rung depends only on proven lower rungs: visual inspection and resistance checks; current-limited rails; clock and reset; JTAG identity; FPGA configuration; minimal LED/counter design; UART; RAM test; then storage and network. At each rung record expected value, instrument, actual value, tolerance, artifact version, and decision.

Preserve a known-good minimal bitstream and host tool. Change one variable at a time and keep rollback cheap. Compare hardware traces with the deterministic simulator at stable interface boundaries rather than expecting cycle-perfect equality across unrelated timing.

### Invariants

- A failed lower rung blocks interpretation of higher-rung tests.
- Every programmed artifact is identified by source revision and reproducible build hash.
- Measurements include units, reference point, expected range, and instrument configuration.
- Reset returns the design to one known state without relying on stale RAM or host state.
- Fault reports separate observation, hypothesis, experiment, and conclusion.

### Safety and limits

Use a current-limited bench supply, insulated probes, eye protection where appropriate, and a written maximum voltage/current/temperature. Stop on excess current, heat, odor, unstable rails, or uncertain grounding. Never probe mains circuitry. The core assignment uses recorded or simulated measurements; physical work is optional and should be supervised.

## Workbook

Given a bringup log showing nominal rails, missing clock, failed JTAG, and random UART bytes, identify which observations are actionable and which are downstream noise. Write the next three safest measurements with expected outcomes. Then predict UART text for three clock/baud mismatch ratios and compare with a capture.

Reorder a deliberately unsafe test plan into a dependency-respecting ladder and explain each moved step.

## Problem set

1. Construct a bringup decision tree for excess current, missing core rail, missing clock, unknown JTAG ID, and configuration failure.
2. Diagnose stable but garbled UART using measured clock, configured divider, framing, voltage, and ground evidence.
3. Design a walking-bit and address-alias RAM test; state which physical faults each pattern can reveal.
4. Explain why changing RTL, constraints, cable, and clock simultaneously destroys diagnostic evidence.
5. Specify a reproducibility record for bitstream, board revision, tool versions, constraints, and host command.
6. Given simulator and logic-analyzer traces with different absolute timing, choose stable events for alignment and locate the first semantic divergence.

## Homework

Produce: (1) a layered bringup plan with stop conditions; (2) a machine-readable result log; (3) simulated fault fixtures for rail, clock, reset, JTAG, RAM, UART, and network layers; (4) a runner that stops at the first failed dependency; (5) one complete diagnostic report; (6) optional hardware evidence gathered under the safety plan.

Acceptance rubric: 25% dependency ordering, 20% precise measurement schema, 20% fault localization, 15% reproducibility/rollback, 10% simulation-to-hardware comparison, and 10% safety discipline. Powering a board without the preflight evidence is an automatic failure of the process rubric.

## Reverse-engineering lens

Treat the board as an unknown system. From measurements, configuration logs, UART bytes, and logic traces, infer the earliest working boundary. Choose the next experiment for maximum information without risking hardware, and update confidence explicitly.

## Exit criteria

- [ ] The bringup ladder proves power, clock, reset, debug, logic, and peripherals in order.
- [ ] Every artifact and measurement is reproducible and attributable.
- [ ] Injected faults are assigned to the correct layer without running unsafe downstream tests.
- [ ] Physical attempts obey current, temperature, grounding, and stop limits.
