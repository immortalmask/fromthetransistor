# Assembler and ISA

## Why this block exists

A processor and assembler are two implementations of one specification. If both are written from shared assumptions instead of checked encodings, they can agree on the same bug. This block makes FTT-16 instruction words, operand ranges, byte order, and rejection rules executable before the CPU depends on them.

## Lecture

FTT-16 is deliberately small: fixed 16-bit instruction words, sixteen 8-bit registers `r0..r15`, and eight operations.

| Assembly | Hex nibbles / fields | Effect |
|---|---|---|
| `halt` | `0000` | stop execution |
| `ldi rD, imm8` | `1 D imm8` | load unsigned 8-bit immediate |
| `add rD, rS` | `2 D S 0` | wrapping 8-bit addition into `rD` |
| `xor rD, rS` | `3 D S 0` | bitwise XOR into `rD` |
| `load rD, addr8` | `4 D addr8` | load one data-memory byte |
| `store rS, addr8` | `5 S addr8` | store one data-memory byte |
| `jmp addr12` | `6 addr12` | set PC to absolute instruction-word index |
| `jz rN, off8` | `7 N off8` | if zero, branch relative to the next word |

The human-readable word `0x122a` is stored as bytes `2a 12` in little-endian memory. Byte order does not reverse the instruction's logical bit fields.

Assembler validation is part of the ISA. Registers stop at `r15`; unsigned 8-bit fields stop at 255; a jump address stops at 4095; branch offsets are signed `-128..127`. `add` and `xor` require the low reserved nibble to be zero when decoding. Unknown opcode or reserved encoding must be rejected, not guessed.

For `jz`, the target rule is:

```text
target_pc = current_pc + 1 + signed(off8)
```

The automated lab assembles or disassembles one exact instruction. A later extension may add source files, comments, labels, and two passes, but it must preserve the word-level oracle.

## Reverse-engineering lens

Disassembly is constrained hypothesis testing. Opcode bits narrow the instruction family; reserved bits reject impossible interpretations; field widths and control-flow targets add consistency checks. When a word decodes syntactically, inspect whether its target lies within the captured program. Valid syntax is not proof that bytes are code.
