# 07.01 JTAG Interface

## Why this block exists

JTAG is often the first observable path into an otherwise silent FPGA. Modeling its TAP state machine and bit order makes the eventual cable driver testable without guessing at USB timing or board wiring.

## Lecture

### Mental model

The Test Access Port samples TMS and TDI on a defined TCK edge and advances through a 16-state controller. Instruction-register scans select a data register; data-register scans shift one bit per clock and update only in the update state. A host operation is therefore an exact TMS/TDI clock transcript, not a high-level “read ID” wish.

Implement a pure TAP model first, including reset by at least five TMS-high clocks, instruction capture/shift/update, BYPASS, and IDCODE. Then implement a shortest-path navigator and a transport interface that can drive the model or a carefully reviewed hardware backend.

### Invariants

- Current TAP state is a pure function of prior state and sampled TMS.
- Shift count and LSB/MSB convention are explicit and identical in host and target.
- An instruction or data value becomes active only on its update transition.
- Reset reaches Test-Logic-Reset from every state within the documented sequence.
- A failed transport cannot be mistaken for a valid all-zero or all-one response.

### Safety and limits

The required lab uses only the simulated TAP. For hardware, verify voltage, ground, connector orientation, and target ownership before driving pins; never assume 5 V tolerance. Cap TCK conservatively and avoid contention with another programmer. The model does not replace vendor configuration algorithms or electrical review.

## Workbook

Starting in an unknown state, predict the TMS sequence to reset, enter Shift-IR, load IDCODE, enter Shift-DR, and read 32 bits. Record state, TMS, TDI, shifted register, TDO, and update event for every clock. Repeat with the bit order reversed and identify the signature in the returned value.

Find two paths from Run-Test/Idle to Shift-DR and justify which the navigator should choose and how it is tested.

## Problem set

1. Complete the full 16-state transition table from the JTAG state diagram and prove the five-high reset property.
2. Derive the clock transcript for shifting an 8-bit instruction LSB-first and exiting without one extra shift.
3. Diagnose IDCODE values of all zeroes, all ones, a one-bit rotation, and a stable unexpected manufacturer field.
4. Explain capture, shift, and update separation and give a bug caused by activating values during shift.
5. Design a transport error result that distinguishes timeout, disconnected target, invalid TDO, and protocol mismatch.
6. Write a differential test comparing the navigator's generated path with breadth-first shortest paths for every state pair.

## Homework

Implement: (1) pure TAP transition model; (2) IR/DR shift registers with explicit bit order; (3) reset and shortest-path navigation; (4) BYPASS and IDCODE instructions; (5) transcript recorder/replayer; (6) fault injection for stuck TMS/TDO, missed clocks, and reversed bits; (7) a transport interface whose hardware implementation remains optional.

## Reverse-engineering lens

Given only TCK/TMS/TDI/TDO samples, reconstruct the TAP states, selected instruction, and shifted value. Locate one missing or extra clock and explain all subsequent misalignment from that earliest event.
