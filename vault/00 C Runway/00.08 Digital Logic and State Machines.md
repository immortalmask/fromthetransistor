# Digital Logic and State Machines

## Why this block exists

Software executes a sequence of completed operations. Synchronous hardware continuously computes candidate next values and captures selected state on clock edges. Without that model, learners often write HDL that looks procedural but builds unintended storage or predicts outputs one cycle early.

## Lecture

Combinational logic maps current inputs to current outputs. A complete truth table defines a Boolean function; gates, muxes, and LUT bits are implementations of that function. Combinational descriptions must assign outputs on every path or they imply remembered state.

Sequential logic stores state. In the course model, registers capture next values on a rising edge. Between edges, combinational logic settles from current state and input. Keep these questions separate:

1. What is the current registered state?
2. What next value does combinational logic compute?
3. Which edge commits it?
4. Which outputs are derived from old versus newly committed state?

An FSM has a finite state set, transition conditions, initial/reset state, and outputs. Illegal states need deliberate behavior. Reset is part of the interface: synchronous reset is sampled at an edge; asynchronous reset can change state outside normal clocking and requires extra physical care.

Use fixed-width unsigned reasoning. A `BITS`-wide counter wraps modulo `2^BITS`; decide whether an output toggles on equality before or after increment. Most off-by-one waveform bugs are specification gaps.

## Workbook

Derive truth tables and four-bit LUT masks using index `a * 2 + b` for AND, OR, XOR, and equality. Predict the mask before calculating it from Boolean expressions.

Trace a two-bit wrapping counter from reset through six rising edges:

| observation | reset | current q | computed next | edge? | observed q |
|---:|---:|---:|---:|---:|---:|

Now design a three-state receiver: `IDLE`, `DATA`, `DONE`. Given inputs `start`, `last`, and `ack`, write a transition table including illegal combinations. Mark which output is Moore-style (state only) and which is Mealy-style (state plus input).

Find the implied latch in pseudocode that assigns `next = IDLE` only when `done` is true. State the missing default.

## Problem set

1. Convert the XOR truth table into a LUT mask under the course index rule.
2. Explain why changing a register with a blocking assignment in clocked HDL can make simulation ordering misleading.
3. A divider toggles output whenever `count == N`. Starting from zero, derive its full period and show where an off-by-one can enter.
4. What should an FSM do when its encoded state is not one of the named states?
5. Distinguish a cycle, a rising edge, a simulator delta/event, and host runtime.

## Homework

Create two representations of the same small controller: a simple software reference step function and an RTL-style state/next-state description on paper or in optional SystemVerilog.

The controller accepts `request`, waits exactly two committed edges, raises `ready`, and returns to idle after `accept`. Deliver:

1. State and transition table, including reset and unexpected input.
2. A cycle trace for normal request, held request, delayed accept, and reset during wait.
3. Assertions for no early `ready`, eventual `ready` under stable clock/reset assumptions, and one-cycle acceptance behavior.
4. A test that would fail if current and next state were confused.

Do not introduce multiple clocks or asynchronous crossings in this exercise.

## Reverse-engineering lens

A waveform is a behavioral trace from which an FSM can be inferred. Look for repeated phases, state-dependent outputs, and events that cause transitions at the next edge rather than immediately. Internal state names are conveniences; the transition relationships are the stronger evidence.
