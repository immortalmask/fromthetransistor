# Simulation and Waveforms

## Why this block exists

Simulation is the course laboratory. It makes internal state observable, reproduces boundary timing, injects failures, and lets anyone run the machine without a board. But an unbounded simulation with ad hoc prints can be as opaque as hardware. This block defines a deterministic harness and first-divergence method.

## Lecture

Separate four notions of time:

- **host runtime:** seconds the simulator takes to execute;
- **simulation time:** timestamps in the hardware model;
- **clock edge:** event at which sequential state commits;
- **cycle:** interval between equivalent clock edges.

A useful testbench controls reset and inputs, advances time deliberately, observes outputs at a stable phase, and asserts the contract. Avoid tests that wait arbitrary host time or rely on a human seeing a waveform.

A waveform is a table whose columns are signals and rows are time points. It is evidence only when signal widths, sampling points, and unknown/uninitialized values are understood. Capture a small cone around the failure: clock, reset, inputs, relevant current/next state, and outputs.

Determinism requires explicit initial state, fixed input sequence, bounded execution, and recorded random seeds. A regression test should fail at the same assertion and cycle. Fault injection—bad frame, delayed readiness, dropped transaction—belongs in the harness, not as nondeterministic environmental hope.

Debug by comparing a reference trace with the implementation and finding the first row that differs. Later rows are consequences. If the first difference is an output, add the smallest internal state needed to determine which transition produced it.

## Workbook

Predict the trace for a one-bit state register with synchronous reset:

| cycle | reset sampled | input d | old q | new q | observed q after edge |
|---:|---:|---:|---:|---:|---:|
| 0 | 1 | 1 | ? | ? | ? |
| 1 | 0 | 1 | ? | ? | ? |
| 2 | 0 | 0 | ? | ? | ? |

Now consider a test that changes `d` at the same simulation timestamp as the rising edge. List two possible ordering interpretations and repair the testbench by driving at an unambiguous phase.

Given expected counter states `0,1,2,3,0` and actual states `0,1,3,0,1`, circle the first divergence. Write two hypotheses that could cause it and one extra signal for discriminating them.

Design an assertion for “after reset is sampled, output is zero at the observation point,” then one for “a request accepted in IDLE leads to BUSY on the next committed state.”

## Problem set

1. Why does viewing a blinking output fail to test the exact divider period?
2. Distinguish an assertion, a stimulus, a fixture, a reference model, and a waveform.
3. A randomized test fails without recording its seed. What evidence is missing?
4. Explain how an uninitialized value can appear to pass a casual `if` test in one simulation and fail elsewhere.
5. A waveform contains 50,000 cycles before failure. Describe a method to reduce the fixture and captured signal set.

## Homework

Create a deterministic simulation-harness design for a wrapping counter or the controller from the runway:

1. Specify reset sequencing and the exact observation phase.
2. Provide a reference step function or hand-derived state table.
3. Generate at least normal, wrap, reset-during-operation, and maximum-parameter cases.
4. Stop after a bounded number of cycles and report the first mismatching cycle.
5. Capture a short annotated waveform or textual equivalent around one deliberate defect.
6. State which internal signals are diagnostic and which are part of the public interface.

If Verilator is available, implement the harness as an optional extension. Otherwise, a cycle-accurate C/Python transition model and textual trace are sufficient for this module.

## Reverse-engineering lens

Treat an unfamiliar waveform as a protocol capture. First identify clock and reset, then stable observation points, repeated phases, and state-correlated outputs. Infer one transition at a time and validate it with a new controlled stimulus. Internal signal names may mislead; timing relationships are stronger evidence.

## Exit criteria

- [ ] Your harness has explicit initialization, observation phase, and execution bound.
- [ ] The same input or seed reproduces the same failure cycle.
- [ ] You identify the first divergence rather than explaining only the final symptom.
- [ ] At least one assertion covers a boundary cycle and one covers reset.
- [ ] You can distinguish host runtime, simulation time, edge, and cycle.
