# Pointers Arrays and Strings

## Why this block exists

Most catastrophic systems bugs are ordinary object-bound or lifetime mistakes crossing an extraordinary boundary. A packet parser, filesystem reader, or syscall copy may receive an address and a claimed length; C will not attach the real bound for you. This block makes every traversal carry its extent and every pointer claim earn its validity.

## Lecture

An array object contains elements. In many expressions its name converts to a pointer to the first element, but the array and pointer remain different types with different `sizeof` results. Pointer arithmetic advances in units of the pointed-to type and is defined only within an array object or one-past its end; the one-past pointer may be compared but not dereferenced.

A C string is a character array containing a terminating zero byte. The terminator is part of the representation but not the logical text length. Untrusted bytes are not a string until a terminator is proven within a known bound. Prefer interfaces shaped like `(const uint8_t *data, size_t length)` or `(char *output, size_t capacity)`.

For a cursor parser, maintain:

```text
base <= cursor <= limit
remaining = limit - cursor
consume(n) only when n <= remaining
```

Subtract within a known object instead of computing an unchecked `cursor + n`. When output may overlap input, state whether overlap is forbidden or implement overlap-aware movement. `memcpy` and `memmove` have different contracts.

`const` on pointed-to data prevents mutation through that access path; it does not make an object globally immutable or extend its lifetime.

## Workbook

Draw object diagrams for:

```c
char a[4] = {'F', 'T', 'T', '\0'};
char *p = a;
const char *q = p + 1;
```

Mark each legal pointer, each dereferenceable pointer, and `a + 4`. Predict `sizeof(a)`, `sizeof(p)`, `strlen(a)`, and the byte read by `q[1]`.

Trace a bounded parser over bytes `03 63 61 74`: the first byte is a length. Use columns `cursor offset`, `remaining`, `requested`, `decision`, and `new offset`. Repeat with `05 63 61 74` and require no partial output mutation.

Given source `[A B C D E]`, simulate copying four bytes from index 0 to index 1 in forward order and backward order. Identify why one order preserves overlapping data.

## Problem set

1. Explain why a function parameter declared `uint8_t data[16]` does not let the function prove that 16 bytes were supplied.
2. Write a safe precondition for consuming `n` bytes from `[cursor, limit)`.
3. A buffer has capacity 8 and must store seven visible characters plus a terminator. Which lengths are acceptable, and what remains unchanged on failure?
4. Why is reading one-past an array invalid even though forming that pointer is valid?
5. Diagnose a loop that calls `strlen` on every iteration over attacker-controlled input.

## Homework

Design and implement a bounded byte-buffer API in C:

1. `buffer_init` establishes storage, length zero, and capacity.
2. `buffer_append` either appends the complete range or changes nothing.
3. `buffer_read_u16_le` advances a caller-owned cursor only on complete input.
4. `buffer_copy_text` writes a terminated string only when capacity is sufficient.
5. All functions define behavior for zero-length input; document whether null pointers are accepted in that case.

Test empty, exact-fit, one-byte-short, embedded zero, overlapping input, and attempted read past end. Run a sanitized build. The artifact includes API contracts, tests, and one object diagram from a failing case.

## Reverse-engineering lens

When reconstructing a function, look for pairs that travel together: pointer plus end, pointer plus count, array base plus index. A bounds check in a predecessor block may dominate a later access; missing that relationship can make safe code look unsafe—or hide a path that bypasses validation. Rename variables by role only after multiple accesses support the hypothesis.
